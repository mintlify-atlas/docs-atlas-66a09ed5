## Submitting a bug report

<Steps>
  <Step title="Open a bug report issue">
    Open an issue using the [Bug report template](https://github.com/ReVanced/revanced-manager/issues/new?assignees=&labels=Bug+report&projects=&template=bug_report.yml&title=bug%3A+) on GitHub.
  </Step>
  <Step title="Fill in the required fields">
    Provide as much detail as possible, including steps to reproduce, expected behavior, and your device and app version.
  </Step>
  <Step title="Include debug logs">
    Export debug logs from **Settings > Advanced > Export debug logs** and attach them to the issue.
  </Step>
</Steps>

## Submitting a feature request

Open an issue using the [Feature request template](https://github.com/ReVanced/revanced-manager/issues/new?assignees=&labels=Feature+request&projects=&template=feature_request.yml&title=feat%3A+) on GitHub. Provide clear motivation for the feature — requests are accepted or rejected at the discretion of the maintainers.

## Contributing code

<Note>
  Development happens on the `dev` branch, not `main`. Always base your work on `dev` and submit pull requests to `dev`.
</Note>

<Steps>
  <Step title="Open an issue to discuss your change">
    Before starting work, open an issue to discuss your proposed change with the maintainers. This helps confirm whether the change is acceptable and avoids wasted effort.
  </Step>
  <Step title="Fork the repository and create a branch">
    Fork the repository on GitHub and create a new branch from the `dev` branch for your changes.
  </Step>
  <Step title="Commit your changes">
    Implement your changes and commit them to your branch.
  </Step>
  <Step title="Submit a pull request to the dev branch">
    Open a pull request targeting the `dev` branch of the main repository. Reference any related issues in the pull request description.
  </Step>
  <Step title="Address review feedback">
    The team will review your pull request and may request changes. Once approved, your changes will be merged into `dev` and included in the next release.
  </Step>
</Steps>
